# AI助手工程规则（Vue3 + JS 专用版）

## ⭐ 全局规则

-   所有回答必须使用中文
-   必须避免重复封装方法，优先复用已有逻辑（hooks / utils / 公共方法）

------------------------------------------------------------------------

## 【阶段一：分析问题】

### 🎯 目标

先搞清楚问题本质，而不是急着写代码。

### ✅ 必须做

#### 1. 判断问题类型

-   Bug（报错 / 死循环 / 数据异常）
-   功能实现
-   重构优化
-   架构设计

#### 2. 定位代码范围

-   优先分析直接相关代码
-   只向上下游扩展 1\~2 层

#### 3. 分析根因

-   数据流（props / emit / store）
-   响应式（ref / reactive / computed / watch）
-   生命周期 / 执行时机

### ⚠️ Vue3专项检查

-   是否误用 ref / reactive
-   是否响应式丢失
-   是否滥用 watch
-   是否存在死循环

### 🔍 复用检查（必须）

-   是否已有相同或类似功能的方法
-   是否可以复用已有 hooks / utils
-   是否存在重复实现逻辑

### ❌ 禁止

-   不分析直接改代码
-   上来就重构
-   忽略已有方法直接重新封装

------------------------------------------------------------------------

## 【阶段二：制定方案】

### ✅ 必须做

-   列出变更文件
-   描述改动内容
-   消除重复逻辑（DRY）

### ⭐ 自动导入判断（强制）

-   检查 vite.config.js 是否有 AutoImport
-   是否存在 auto-imports.d.ts

规则： - 已启用 → 不要手动 import - 未启用 → 必须 import

### ⭐ 复用优先（强制）

-   优先复用已有方法
-   若必须新增方法：
    -   说明为什么不能复用
    -   放入合理位置（hooks / utils）
-   禁止重复封装相同逻辑

### ⭐ 多方案决策

-   给出 A/B 方案
-   对比优缺点
-   推荐最佳方案

### ⭐ 影响范围评估

-   是否影响其他模块
-   是否存在副作用

------------------------------------------------------------------------

## 【阶段三：执行方案】

### ✅ 必须做

-   提供完整可运行代码
-   保持 Vue3 + JS 风格

### ⭐ 自动导入适配

#### 已开启自动导入

``` js
const count = ref(0)
```

#### 未开启自动导入

``` js
import { ref } from 'vue'
const count = ref(0)
```

### ⭐ 代码质量要求

-   不引入无关依赖
-   不修改无关代码
-   保持代码清晰
-   不重复封装已有逻辑

### ❌ 禁止

-   引入未使用 API
-   忽略自动导入机制
-   重复实现已有方法

------------------------------------------------------------------------

## 🔥 核心原则

1.  先分析，再动手
2.  控制影响范围
3.  优先 computed，减少 watch
4.  写代码前先判断自动导入
5.  优先复用，避免重复封装
6.  保持代码简单
